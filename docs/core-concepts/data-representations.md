# Data Representations & Configuration Specification

This document details the primary Kotlin data classes and YAML configurations that define the data space of **TaxoArena**. It provides a clear reference for the representations of nodes, embeddings, pairwise stats, rating state, leaderboards, and execution settings.

---

## 1. Embeddings & Vectors

### `Embedding`
Represents a text query transformed into a high-dimensional vector $x \in \mathbb{R}^d$. It carries both the raw query text and its distilled semantic signature, along with ground-truth category metadata.

*   **Key Fields**:
    *   `rawText: String`: The raw, verbose query from the dataset (e.g., MMLU-Pro question with all context).
    *   `distilledText: String`: The distilled keyword signature used to denoise the embedding space.
    *   `values: FloatArray`: The raw high-dimensional vector (typically $d=4096$ from Qwen3-Embedding).
    *   `groundTruthCategory: String`: The MMLU-Pro ground-truth label (e.g., "computer_science"). Passed from taxonomy creation to survive downstream relabeling. Used for validation metrics.
    *   `queryId: Int`: Transient integer ID mapping to the source database.
*   **Key Operations**:
    *   `toDoubleArray()`: Promotes `FloatArray` to `DoubleArray` to ensure high-precision calculations (e.g., in covariance updates).
    *   `projectTo(targetDim: Int)`: Slices and L2-normalizes the vector to project it into a lower dimension for depth-varying statistical modeling.

## 2. Taxonomy Nodes

### `GraphNode`
The core topological unit in the subject DAG taxonomy. A node represents a knowledge domain and houses the mathematical modeling parameters.

*   **Key Fields**:
    *   `id: String`: Unique UUID.
    *   `label: String?`: Concise natural language label generated by the LLM (e.g., "Quantum Mechanics").
    *   `depth: Int`: Topological distance from the Root (0-indexed).
    *   `children: MutableSet<GraphNode>`: Set of primary tree child nodes (defines tree structure and `isLeaf`).
    *   `crossLinkChildren: MutableSet<GraphNode>`: Secondary polyhierarchical child edges (does not affect `isLeaf`).
    *   `parents: MutableSet<GraphNode>`: Unified list of parent nodes (both tree parents and cross-linked parents).
    *   `queries: MutableList<Embedding>`:
        *   If `isLeaf == true`: Holds all queries belonging to this specific terminal leaf domain.
        *   If `isLeaf == false`: Holds parent residual queries (outliers that failed to trickle down).
    *   `sliceDim: Int`: Projection dimension $d(depth)$.
    *   **vMF Modeling**:
        *   `vmfMu: FloatArray`: Unit-norm mean direction vector $\mu \in S^{d-1}$.
        *   `vmfKappa: Double`: Concentration parameter $\kappa$ (bias-corrected).
        *   `vmfLogNormalizer: Double`: Precomputed log-normalizer $\ln C_d(\kappa)$.
    *   **NiW Modeling**:
        *   `niwM0: FloatArray`: Mean vector $\mu_N$ of the Normal-Inverse-Wishart posterior.
        *   `niwKappa0: Double`: Posterior pseudocount parameter $\kappa_N$.
        *   `niwNu0: Double`: Posterior degrees of freedom $\nu_N$.
        *   `niwLambda: FloatArray`: Diagonal of the posterior covariance scale matrix $\Lambda_N$.
    *   **Judge Profiles**:
        *   `judgePrompt: String?`: System instructions for LLM pairwise grading.
        *   `judgeRubric: String?`: Grading rubric generated from domain axioms.
*   **Key Properties**:
    *   `isLeaf: Boolean`: Evaluates as `children.isEmpty()` (ignores `crossLinkChildren`).
    *   `treeParentId: String?`: Tracks the immutable parent assigned at creation time.

## 3. Benchmarking & Ranking Models

### `NodePairStats`
Accumulates pairwise contest outcomes between model pairs at a specific taxonomy node.
*   **Key Fields**:
    *   `nodeId: String`: The UUID of the evaluating node.
    *   `modelA: String` & `modelB: String`: Names of the competing models.
    *   `winsA: Double` & `winsB: Double`: Accumulated wins. A win adds `1.0`, a tie adds `0.5` to both.
    *   `ties: Int`: Number of contested matches that resulted in a tie (no model preferred).
    *   `totalComparisons: Int`: Total matchups evaluated.
    *   `positionFlips: Int`: Counts how often swapping evaluation order changed the win preference (order bias indicator).

### `NodeBtState`
Stores the fitted Bradley-Terry parameter state for a node.
*   **Key Fields**:
    *   `nodeId: String`: Evaluating node.
    *   `btScores: Map<String, Double>`: Map of model names to log-strength parameters $\beta$.
    *   `stdErrors: Map<String, Double>`: Standard error of the log-strength parameters.
    *   `fitVersion: Int`: Monotonically increasing revision counter.
    *   `totalComparisons: Int`: Total comparisons feeding the fit.
    *   `lastFitAt: Long`: Epoch timestamp.

### `AgentRating`
Stores a model's local or global rating (OpenSkill baseline).
*   **Key Fields**:
    *   `agentName: String`: Model name.
    *   `domain: String`: Taxonomy node identifier (or "global").
    *   `mu: Double`: Mean skill rating.
    *   `sigma: Double`: Uncertainty standard deviation.
    *   `active: Boolean`: Indicates if the model is active in matchmaking.

## 4. System Settings & Configuration

The configuration parameters are managed via a nested YAML setup (mapped to `TaxonomyConfig` in Kotlin). Below is a comprehensive specification of these settings:

```yaml
taxonomy:
  formalism:
    minClusterSize: 20            # Minimum queries required to keep a node or trigger a split ($N_{min}$)
    maxDepth: 7                  # Maximum allowable depth of the taxonomy DAG ($D_{max}$)
    separationEpsilon: 0.05      # JS-divergence distinctness threshold ($\epsilon$) for sibling merging
    assignmentGap: 0.15          # Trickle routing assignment boundary in nats
    cosineTau: 0.5               # Temperature ($\tau$) for softmax routing probabilities
    emaAlpha: 0.3                # Decay rate ($\alpha_{EMA}$) for vMF concentration parameters
    biasCorrectionThreshold: 5.0 # Ratio $d/N$ above which kappa bias correction is applied
  llm:
    labelingModel: "gemini-2.5-flash"  # Model used for generating domain-specialized labels
    judgeModel: "gemini-2.5-pro"        # Model instantiated for pairwise trajectory evaluation
    apiBaseUrl: "https://api.provider.com"
  execution:
    enableLabeling: true         # Enable/disable cluster naming via LLM
    enableLiveLabeling: false    # Generate labels during execution or post-pass
    llmParallelism: 8            # Maximum concurrent requests to the LLM API
```
